# HardwareController Composition Evidence

## Test Date: 2025-03-13

## Components Composed:

### 1. STLinkManager (from stlink_manager.py)
- Used for: Hardware lifecycle control
- Methods used:
  - is_connected() -> tuple[bool, str]
  - reset() -> tuple[bool, str]
- Location: util/stlink_manager.py

### 2. UARTChannel (from uart_channel.py)
- Used for: Serial communication over UART3 VCP
- Methods used:
  - open()
  - close()
  - send(data)
  - readline(timeout)
  - query(command, timeout)
  - drain()
  - wait_for_ready(timeout)
- Location: util/uart_channel.py

### 3. CommandMapper (from protocol.py)
- Used for: Protocol translation (USB/GUI → TestMode)
- Methods used:
  - map_usb_command(data) -> str
  - map_gui_command(command) -> str
  - parse_response(response) -> bytes
- Location: util/protocol.py

## Composition Pattern:
```python
class HardwareController(BaseController):
    def __init__(self, port='/dev/ttyACM0', baudrate=115200, stflash_path='st-flash'):
        self._stlink = STLinkManager(st_flash_path=stflash_path)
        self._uart = UARTChannel(port=port, baudrate=baudrate)
        self._mapper = CommandMapper()
```

## Design Principle:
- Compose, don't duplicate
- Each component has a single responsibility
- HardwareController orchestrates the components
